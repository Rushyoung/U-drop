# 后端架构与数据库设计

## 系统架构
- **logger**: 负责写入和维护日志文件。
- **fastapi**: 提供 RESTful 接口。
- **service**: 处理业务逻辑（认证、时间线、文件处理）。
- **repository**: 封装 SQLite (udrop.db) 的数据读写。
- **data_manager**: 兼容入口，保留给历史调用。
- **client**: 网页端 (Vue3/WASM) 或 安卓端 (Kotlin/JNI)。
- **C-Core**: 通过 ctypes/FFI 调用，处理哈希计算和图片缩略图。

## 数据库设计 (udrop.db)

### 1. users (用户表)
| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| uuid | str | PK | 用户唯一标识 |
| account | str | Unique | 登录账号 |
| password_hash | str | - | 加盐后的哈希值 |
| created_at | int | - | 注册时间戳 |

### 2. devices (设备表)
| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| device_id | str | PK | 设备唯一标识 |
| user_uuid | str | FK -> users.uuid | 所属用户 |
| device_type | int | - | 0:Web, 1:Android |
| device_name | str | - | 设备名称 (如: Pixel 6) |
| last_seen | int | - | 最后活跃时间 |

### 3. messages (消息/时间线表)
| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| id | int | PK, Increment | 消息 ID |
| sender_uuid | str | FK -> users.uuid | 发送者 |
| device_id | str | FK -> devices.device_id | 发送设备 |
| type | int | - | 0:文本, 1:图片, 2:文件 |
| content | text | - | 文本内容或消息描述 |
| file_hash | str | FK -> file_info.full_hash | 关联文件 (可选) |
| timestamp | int | Index | 发送时间戳 |
| is_deleted | int | Default 0 | 软删除标记 (0:正常, 1:已删除) |

### 4. file_info (文件信息表)
| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| full_hash | str | PK | 全量哈希 (Blake3) |
| sparse_hash | str | Index | 快速哈希 (头中尾各4KB MD5) |
| file_name | str | - | 原始文件名 |
| file_size | bigint | - | 文件大小 (Bytes) |
| mime_type | str | - | 文件类型 |

| thumbnail_url | str | - | 预览图访问路径 (图片类文件) |
| refer_count | int | Default 1 | 引用计数 (用于物理文件清理) |

## 时间线读取与删除同步（实现约定）
- 时间线读取使用“当前消息的 id”分段拉取：
	- 首次拉取：获取最近一屏
	- 查看更早消息：带上当前最旧消息 id
	- 拉取新消息：带上当前最新消息 id
- 消息删除采用软删除：设置 `is_deleted = 1`。
- 服务端在消息变更后下发 `REFRESH_TIMELINE` 信号，客户端再调用同步接口拉取变更并更新本地列表。

### 8. upload_tasks (上传任务表)
上传任务当前计划为内存态维护（例如进度存 dict）。

| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| upload_id | str | PK | 上传任务 ID |
| user_uuid | str | FK -> users.uuid | 所属用户 |
| temp_path | str | - | 临时文件路径 |
| received_size | bigint | Default 0 | 已接收大小 |
| total_size | bigint | - | 文件总大小 |
| created_at | int | - | 创建时间戳 |

说明
- 表结构保留为预留/备份方案
- 实际运行时以内存态为准
